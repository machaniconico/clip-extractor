🔑 clip-extractor はじめてのセットアップガイド

「Gemini APIキー」と「credentials.json」——2つの“鍵”のもらい方を、パソコンが苦手な方にもわかるように図解つきで説明します。
どちらも 無料 で、クレジットカードの登録は不要 です。

0. さいしょに読んでください — 「鍵」は2種類あります

clip-extractor が Google のサービスを使うためには、Google から「鍵」をもらってくる必要があります。 鍵は 2種類 あって、用途がまったく違います。名前が似ていて混同しやすいので、まずここだけ押さえてください。

あなたに必要なのはどっち？

やりたいこと鍵A（Gemini APIキー）鍵B（credentials.json）
動画から切り抜きを自動で作りたい（基本機能）✅ 必要 ※不要
切り抜きのタイムスタンプを YouTube の概要欄に自動で書き込みたい✅ 必要 ※✅ 必要
できた動画を Google Drive に自動アップロードしたい✅ 必要 ※✅ 必要

※ AI に Claude（CLI）や OpenAI を使う設定にしている場合、鍵Aの代わりにそちらの準備が必要です。とくにこだわりがなければ「無料枠があってクレカ登録不要」の Gemini がおすすめです。

💰 安心してください：このガイドに出てくる作業は、個人で使う範囲なら 全部無料 です。クレジットカードの登録画面は一度も出てきません。

Part 1：Gemini APIキーをもらう ⏱ 約2分 無料・クレカ不要

AI（Gemini）に「動画のどこが面白いか」を分析してもらうための鍵です。Google アカウント（Gmail のアカウント）があれば 2分でもらえます。

1

Google AI Studio のページを開く

ブラウザ（Edge や Chrome）の一番上にある「アドレスバー」に、次の文字をそのまま打ち込んで Enter キーを押します（この文書をパソコンで見ているなら、クリックするだけで開きます）。

https://aistudio.google.com/apikey
2

Google アカウントでログインする

ログイン画面が出たら、ふだん使っている Google アカウント（Gmail のアドレス）でログインします。

⚠️ 会社や学校の Google アカウントは避けてください。組織の設定でこのページがブロックされていることがあります。個人の Gmail アカウントを使うのが確実です。
3

利用規約に同意する（初回だけ）

初めて開いたときは、利用規約への同意を求める画面が出ます。内容を確認してチェックを入れ、進んでください。

💡 このとき「デフォルトプロジェクト」と一緒に最初のキーが自動で作られることがあります。画面にすでに AIza… で始まる文字列が表示されていたら、それをそのまま使って OK です（その場合はステップ 6 へ）。
4

「＋ APIキーを作成」ボタンを押す

5

プロジェクトを聞かれたら「新規作成」でOK

「どのプロジェクトで作りますか？」という選択が出ることがあります。よく分からなければ新規作成を選んでください。名前を付けるだけで、難しい知識は一切いりません。

6

できあがったキーをコピーする

AIza から始まる長い文字列が表示されます。横にあるコピーボタン（📋 のようなアイコン）を押してコピーします。

🔒 このキーはパスワードと同じです。人に見せたり、SNS やインターネットに書き込んだりしないでください。
7

clip-extractor に貼り付けて保存する

  1. clip-extractor を起動します（Windows なら Clip Extractor.bat をダブルクリック）
  2. ブラウザに表示された画面で 「Settings / 設定」タブ を開きます
  3. 「APIキー」という入力欄に、コピーしたキーを貼り付け（右クリック →「貼り付け」）
  4. すぐ下の 「💾 このキーを保存」ボタンを押します
✅ これで Part 1 は完了です！ 切り抜きの自動生成だけが目的なら、もう使い始められます。

Part 1 のよくある質問

質問答え
お金はかかりますか？かかりません。無料枠の範囲で使えて、クレジットカードの登録も不要です。
「429」というエラーが出ました無料枠の「回数制限」に一時的に達しただけです。数分〜しばらく待ってもう一度実行すれば直ります。
どのモデルを選べばいい？最初から選ばれている gemini-2.5-flash のままで OK です。
前に作ったキーが急に使えなくなりました2026年のセキュリティ強化で古いキーが順次無効化されています。同じページでキーを作り直すのが最速です。

Part 2：credentials.json をもらう ⏱ 約10分 無料・クレカ不要

💡 この Part が必要なのは「YouTube 概要欄への自動書き込み」や「Google Drive への自動アップロード」を使いたい人だけです。切り抜き生成だけなら飛ばして構いません。あとから必要になったときに戻ってくれば OK です。

Part 1 より手順は多いですが、1つずつ進めば必ずできます。最初の1回だけの作業で、次からは何もしなくて大丈夫です。全体の流れはこうです：

⚠️ Google の画面は時々デザインが変わります。この先の図は「だいたいこういう配置」という模式図です。ボタンの文言を頼りに探してください。この手順は 2026 年時点の新画面（Google Auth Platform）に対応しています。
1

Google Cloud Console を開く

https://console.cloud.google.com/

Part 1 と同じ Google アカウントでログインしてください。

⚠️ トップ画面に「Create Gemini API key」などのボタンが見えても押さないでください。それは Part 1 でもう済んでいる別の作業です。これから作るのは「OAuth クライアント」という別の種類の鍵です。
2

「プロジェクト」（作業場所）を作る

Google Cloud では、何をするにもまず「プロジェクト」という入れ物（作業場所）を作ります。

  1. 画面最上部の青いバーにある「Google Cloud」ロゴのすぐ右のプルダウン（「プロジェクトを選択」などと表示されている部分）をクリック
  2. 開いたウィンドウの右上 「＋ 新しいプロジェクト」 をクリック
  3. プロジェクト名に clip-extractor と入力（好きな名前でOK）。「組織」は「組織なし」のままで OK
  4. 青い 「作成」 ボタンを押す → 数秒で完了します
🚨 いちばん多いミス：作ったあと、そのプロジェクトに切り替わっていないことがあります。上の図の場所が 「clip-extractor」という表示になっているか必ず確認してください。なっていなければ、同じ場所をクリックして一覧から clip-extractor を選びます。ここがズレていると、この先の作業がすべて別の場所で行われてしまいます。
3

使う機能のスイッチを入れる（API の有効化）

「うちのプロジェクトは YouTube の機能を使います」というスイッチを入れます。次の URL を開いてください：

https://console.cloud.google.com/apis/library
  1. 検索欄に YouTube Data API v3 と入力
  2. 出てきたカードをクリック
  3. 青い 「有効にする」（英語なら ENABLE）ボタンを押す
  4. 5〜30 秒待つと、緑のチェック「✓ API が有効です」が表示されます
⚠️ そっくりさんに注意！ 検索すると似た名前が3つ出てきます：
✅ YouTube Data API v3 ← これが正解
❌ YouTube Analytics API（視聴分析用・別物）
❌ YouTube Reporting API（レポート用・別物）
💡 有効化したあとの画面に「管理」「この API を試す」というボタンが出ますが、どちらも押さなくて大丈夫です。押すと関係ない画面に飛んで迷子になります。そのまま次のステップへ進みましょう。

Google Drive への自動アップロードも使いたい場合は、同じ手順で Google Drive API も検索して有効にしてください（概要欄の自動書き込みだけなら不要です）。

4

アプリ情報を登録する（初回だけのかんたんな質問）

次の URL を開きます：

https://console.cloud.google.com/auth/overview

「Google Auth Platform はまだ構成されていません」と表示されるので、「使ってみる」（英語なら Get started）を押します。4つの質問に答えるだけの画面（ウィザード）が始まります：

画面入力すること
1. アプリ情報アプリ名：clip-extractor／サポートメール：自分の Gmail を選択 → 次へ
2. 対象「外部」を選ぶ（個人アカウントではこれしか選べません）→ 次へ
3. 連絡先情報自分のメールアドレスを入力 → 次へ
4. 完了ポリシー同意にチェック → 「作成」
5

★最重要★ 自分を「テストユーザー」に登録する

🚨 ここが Part 2 全体でいちばん忘れやすく、忘れると後で必ずエラーになる場所です。前のステップのウィザードには含まれていないので、自分で登録する必要があります。
  1. 次の URL を開く（または画面左側のメニューで「対象」をクリック）：
    https://console.cloud.google.com/auth/audience
  2. 画面の中ほどにある 「テストユーザー」 セクションを探す
  3. 「＋ ユーザーを追加」（+ Add users）をクリック
  4. 自分の Gmail アドレスを入力して「保存」
6

鍵を作る（OAuth クライアントの作成）

次の URL を開きます（または左メニューの「クライアント」）：

https://console.cloud.google.com/auth/clients
  1. ページ上部の 「＋ クライアントを作成」 をクリック
  2. 「アプリケーションの種類」で ★「デスクトップ アプリ」★ を選ぶ
  3. 名前は clip-extractor desktop など好きな名前で OK
  4. 「作成」 ボタンを押す
🚨 種類はかならず「デスクトップ アプリ」を選んでください。「ウェブ アプリケーション」を選ぶと、あとで redirect_uri_mismatch というエラーになります。
7

JSON ファイルをダウンロードする（★このタイミングだけ！）

作成すると「OAuth クライアントを作成しました」というウィンドウが出ます。

🚨 このウィンドウを閉じる前に、必ず「JSON をダウンロード」を押してください。
2025年6月の仕様変更で、完全な鍵ファイルを保存できるのはこの直後の1回だけになりました。閉じてしまうと、同じファイルは二度とダウンロードできません（その場合の対処はトラブルシューティングへ。作り直せば OK なので焦らなくて大丈夫です）。
🔒 このファイルもパスワードと同じです。人に渡したり、インターネットにアップロードしたりしないでください。
8

clip-extractor にファイルを渡して、認証する

いちばん簡単な方法（ドラッグ＆ドロップ）：

  1. clip-extractor を起動して 「Settings / 設定」タブ を開く
  2. 「YouTube API 認証」セクションにある 「credentials.json（ドラッグ＆ドロップ可）」という枠に、さっきダウンロードした JSON ファイル（通常は「ダウンロード」フォルダにあります）をマウスでつかんで放り込む
  3. 緑の成功メッセージが出たら配置完了
  4. 同じセクションの 「認証する」 ボタンを押す
  5. ブラウザに Google の確認画面が開くので、ステップ 5 で登録した Gmail アカウントを選んで許可する
💡 途中で「このアプリは Google で確認されていません」という警告が出ますが、自分で作ったアプリなので問題ありません。「続行」を押して進んでください。
✅ 画面が「認証済み」になったら全部完了です！おつかれさまでした 🎉
以降は鍵が自動で更新されるので、この作業を繰り返す必要はありません。

3. こまったときは（トラブルシューティング）

❌「アクセスをブロック」「エラー 403: access_denied」と出る

原因の第1位：テストユーザーの登録忘れ（Part 2 のステップ 5）です。

  1. console.cloud.google.com/auth/audience を開く
  2. 「テストユーザー」セクションで「＋ ユーザーを追加」→ 自分の Gmail を入力して保存
  3. 1〜2分待ってから、clip-extractor の「認証する」をもう一度押す

❌「redirect_uri_mismatch」というエラーが出る

鍵を作るとき（ステップ 6）に「ウェブ アプリケーション」を選んでしまったのが原因です。クライアント一覧でそのクライアントを削除し、「デスクトップ アプリ」を選び直して JSON を再ダウンロードしてください。

❌ JSON をダウンロードし忘れた／ファイルをなくした

作り直せば大丈夫です（無料・何度でも可）。方法は2つ：

  • 方法1：クライアント一覧で該当の名前をクリック →「クライアント シークレット」欄でシークレットを追加（リセット）→ 新しい JSON をその場でダウンロード
  • 方法2：該当クライアントを削除して、ステップ 6 からやり直し

❌ 数日たったら「再認証してください」と言われる

アプリが「テスト」モードのままだと、認証は 7日で期限切れになる仕様です。

  • 手軽な対処：Settings タブで「認証する」をもう一度押すだけ
  • 恒久対処：「対象」ページの公開ステータスで「アプリを公開」を押して本番モードにする（自分用アプリなら審査は通常不要です）

❌ Gemini で「429」エラーが出る

無料枠の回数制限に一時的に達しただけです。数分〜しばらく待って再実行してください。壊れたわけではありません。

4. ことばのミニ辞典

ことばざっくりした意味
API（エーピーアイ）アプリ同士が会話するための「窓口」。clip-extractor はこの窓口を通じて Google のサービスにお願いごとをします。
APIキー窓口で見せる「会員証」。AIza… のような長い文字列で、これを見せると Google が「この人は登録済みだな」と分かってくれます。
OAuth（オーオース）「私の YouTube アカウントを、このアプリに操作させていいですよ」と Google に伝える許可のしくみ。パスワードそのものをアプリに渡さずに済む、安全な方法です。
credentials.jsonOAuth を使うための「アプリの身分証明ファイル」。Part 2 で作ってダウンロードするファイルです。
JSON（ジェイソン）コンピュータ用のデータの書き方の名前。「.json」で終わるファイルは、設定や情報がこの書き方で入っています。中身を編集する必要はありません。
プロジェクトGoogle Cloud の中の「作業フォルダ」のようなもの。この中に設定や鍵がまとまります。
クォータ「1日に使える回数の上限」。YouTube の窓口は1日 10,000 ポイントぶん無料で使えて、概要欄の書き込み1回で 50 ポイント使います（＝1日約200本ぶん。ふつうの使い方でなくなることはまずありません）。

もっと詳しい情報（コマンドラインでの操作・保存先のパスなど）は、同じフォルダの CREDENTIALS_SETUP.txt と README.md にあります。
このガイドは 2026 年 7 月時点の Google の画面に基づいています。
